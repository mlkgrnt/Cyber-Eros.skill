# LoreDistiller 设定蒸馏协议

## 概述

本协议旨在将冗长的小说、同人文或背景设定集，提纯为 LLM 友好的高密度数据块。

不仅提取硬设定，还支持**剧情提炼**与**文风克隆**。协议具备环境感知能力，可在终端/Agent 环境下自动完成文件目录构建。

## 核心工作流

INTAKE → TRIAGE（AI预判与人工裁决）→ EXTRACT（多维深度解析与提纯）→ EXPORT（环境自适应文件生成）

## 阶段一：文本接收与维度定制 (INTAKE Phase)

当用户触发技能并提供长文本后，LLM **必须暂停**，不要立刻生成内容，而是先向用户发送以下引导语：

"收到文本！为了精准提纯，请告诉我你希望蒸馏哪些维度？

[ ] A. 核心剧情与时间线 (Plot Distillation) —— 提取事件节点、核心矛盾与剧情走向

[ ] B. 角色设定 (Character Info) —— 提取人设、外貌、性格与动机

[ ] C. 人物关系网络 (Relationships) —— 提取表面关系与隐性情感张力

[ ] D. 世界观与背景设定 (Worldbuilding) —— 提取地点、规则与时代质感

[ ] E. 文笔与文风特征 (Style Cloning) —— 提取作者的惯用词汇、感官偏好与叙事节奏

直接回复字母组合（如 ABCE 或 全选），我立刻开始进行文本扫描～"

## 阶段二：智能预判与裁决 (TRIAGE Phase)

收到维度选择后，LLM **在内部进行快速评估**。

拦截可能污染设定的边缘元素，并向用户输出一份「瘦身建议」：

"文本扫描完毕！为了保证未来扮演的专注度，我做了一个初步的权重预判：

🟢 **高权重（建议保留）**：

- **核心角色/设定**：[列出 2-3 个核心要素]
- **主线剧情/风格特征**：[列出 1-2 条，如"复仇主线"、"冷峻细腻的感官描写"]

🟡 **低权重（建议剔除）**：

- **边缘元素**：[列出 3-5 个无互动的背景板人物、与主线无关的支线]

你看看这个预判方案可以吗？直接回复'同意'，或者告诉我'保留 [某某]'、'剔除 [某某]'。确认后我马上生成最终结构化文件！"

## 阶段三：深度解析 (EXTRACT Phase)

在用户确认名单后，LLM 按照最终名单进行深度提取。

**核心规则：**

1. 彻底屏蔽被标记为"剔除"的人物、地点和事件。
2. **剧情蒸馏**：剥离灌水内容，只留核心冲突、转折点和名场面。
3. **文风蒸馏**：分析句子长度、高频感官词、情绪基调。

## 阶段四：解耦生成与文件写入 (EXPORT Phase)

**⚠️ 最高指令：严格遵循 specs/ 输出规范（LLM 必读）：**

本技能文件夹 `specs/` 目录下包含所有输出文件的权威模板。LLM **必须**严格按照以下规则执行：

- **模板强制匹配**：每个输出文件的内容必须与 `specs/` 中对应模板的**字段结构完全一致**，不得增删字段、不得改变嵌套层级。
- **JSON 严格格式**：JSON 文件必须输出标准格式，**不允许带有尾部逗号 (Trailing comma)**，键名必须与模板完全一致（如 `"intimacy_base"` 不可改为 `"intimacy"`）。
- **时间戳命名**：所有文件名强制附加当前现实时间戳，格式为 `[文件名]-YYYY-MM-DD.[扩展名]`（例如 `Character_Info-2026-05-01.md`）。
- **环境自适应**：Agent 环境 → 静默调用文件系统工具直接写入文件；Web 环境 → 以 Markdown 代码块输出。
- **剧情时间节点设置**: 默认时间节点应设为主线剧情开始时。

### 1. 角色档案 — 严格遵循 `specs/Character_Info.md` 模板

### 2. 世界观背景 — 严格遵循 `specs/Background_Info.json` 模板

### 3. 人物关系网络 — 严格遵循 `specs/Relationships.json` 模板

### 4. 剧情蒸馏 — 严格遵循 `specs/Plot_Timeline.md` 模板

### 5. 文风克隆配置 — 严格遵循 `specs/Writing_Style.md` 模板

## 结束语规范

执行完文件输出（或文件写入）后，向用户发送：

"💽 设定已全部蒸馏完毕！

包含：角色档案、世界观、人物关系、剧情主线以及文风克隆配置。

现在可以直接呼叫 /eros 协议，让我们带着原著的灵魂开始吧～"
